- CrossEncoder 在验证集准确率、F1 全面优于 BiEncoder，
- BiEncoder 文本交互仅在相似度计算阶段；CrossEncoder 输入拼接后深度交互，细粒度语义匹配能力更强，匹配类任务天然精度更高。
- LLM SFT 是生成式自回归模型，适合文本生成，不适合处理文本匹配任务


| 对比维度             | BiEncoder Cosine 分类损失 | BiEncoder Triplet 三元组损失 |
| ---------------- | --------------------- | ----------------------- |
| 训练损失 train_loss  | 0.1525                | 0.0259                  |
| 验证准确率 val_acc    | 0.8444                | 0.8342                  |
| 验证 F1 val_f1     | 0.8443                | 0.8340                  |
| 最优分类阈值 threshold | 0.67                  | 0.57                    |
| 单 Epoch 平均耗时     | ≈1244s                | ≈526s                   |

- Cosine 方案是有监督二分类任务：直接优化 “匹配 / 不匹配” 标签，模型学习目标和线下评价指标 Acc/F1 完全对齐，优化方向和评估标准一致；
- Triplet 损失优化目标是相对距离：只约束正样本距离 < 负样本距离 + margin，不直接优化分类边界，无法精准拟合分类打分阈值，最终分类指标上限更低；
- Triplet 擅长拉开正负样本整体距离，但不擅长学习精细的分类分割线。
- Triplet 损失数值极低，收敛更快
