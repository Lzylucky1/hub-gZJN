## 1. guided_choice 约束解码

### 1.1 原理

- **run_without_guided**：完全依赖模型自觉去遵守 system prompt 里的指令。
- **run_with_guided_choice**：

  ```python
  extra_body={"guided_choice": INTENT_CHOICES},   # ★ 关键参数
  ```

  这是 vLLM（或类似推理引擎）提供的 **guided decoding / structured output** 功能。原理是：在生成**每一个 token 的时候**，引擎会在底层维护一个有限状态机（FSM），根据 `INTENT_CHOICES` 里允许的选项动态计算出"当前位置合法的 token 集合"，然后把不合法 token 的 logits 直接置为 `-inf`（相当于屏蔽掉），再做采样。

### 1.2 结果判定说明

`without-guided` 只有三种情况：

1. 答案正确且格式正确
2. 格式正确但答案不正确
3. 答案不正确（格式也不正确）

### 1.3 测试结果

**Choices**：`['查股价', '查财报', '查新闻', '对比分析', '其他']`

| 平安银行去年（2024）的 ROE | ✓ `{"company": "平安银行", "year": 2024, "metric": "ROE"}` | ✓ 同上 | ✓ 同上 |

| 2023 宁德时代经营现金流 | ✓ `{"company": "宁德时代", "year": 2023, "metric": "经营现金流"}` | ✓ 同上 | ✓ 同上 |
| 问一下比亚迪 2024 的总资产规模 | ✓ `{"company": "比亚迪", "year": 2024, "metric": "总资产"}` | ✓ 同上 | ✓ 同上 |

| 隆基绿能 22 年 roe | ✗ `{"company": "隆基绿能", "year": 22, "metric": "ROE"}` | ✗ 同上 | ✗ 同上 |

**9 条测试结果汇总**

| 指标 | 裸 prompt | response_format | guided_json |
|---|---|---|---|
| 合法 JSON | 9/9 (100%) | 9/9 (100%) | 9/9 (100%) |
| 字段齐全 | 9/9 (100%) | 9/9 (100%) | 9/9 (100%) |
| year 在 2015~2025 | 8/9 (89%) | 8/9 (89%) | 8/9 (89%) |

**结论**：`response_format` 只保证是 JSON，不保证字段名、类型、枚举正确；`guided_json` 是唯一 100% 保证 schema 合法的方式。

## 3. 标准化（guided_regex）

### 3.1 任务 1：日期标准化 → `YYYY-MM-DD`

正则：`\d{4}-\d{2}-\d{2}`

| 输入 | 裸 prompt | guided_regex |
|---|---|---|
| 2024年5月12日 | ✓ 2024-05-12 | ✓ 2024-05-12 |
| 2023/12/1 下午开会 | ✗ 2023-12-01 14:00:00 | ✗ 2023-12-01 14:00:00 |
| 三月三号我去北京 | ✓ 2017-03-03 | ✓ 2017-03-03 |
| 2024.11.30 是截止日期 | ✓ 2024-11-30 | ✓ 2024-11-30 |
| 明天（假设今天是2026-05-11） | ✓ 2027-04-12 | ✓ 2027-04-12 |
| 2024 年 10 月的第一天 | ✓ 2024-10-01 | ✓ 2024-10-01 |

格式合法率：裸 prompt 5/6 (83%)　｜　guided_regex 5/6 (83%)

### 3.2 任务 2：A 股代码抽取 → 6 位数字

正则：`\d{6}`

| 输入 | 裸 prompt | guided_regex |
|---|---|---|
| 帮我查 600000 浦发银行 | ✗ 600000浦发银行股票代码是：600000… | ✗ 600000浦发银行股票代码是：600000.0001 |
| code: 000001 平安银行 | ✗ 平安银行600001 | ✗ 平安银行600001 |
| 茅台的代码是 600519 | ✓ 600519 | ✓ 600519 |
| 六零零五一九 | ✗ 059 | ✗ 059 |
| 股票代码：300750（宁德时代） | ✓ 300750 | ✓ 300750 |

格式合法率：裸 prompt 2/5 (40%)　｜　guided_regex 2/5 (40%)

**结论**：`guided_regex` 保证下游解析器永远能拿到合法输入，特别适合日期/电话/代码/邮编等有严格格式的字段。

## 4. response_format（OpenAI 标准 JSON 模式）

| Prompt | raw | json_object |
|---|---|---|
| 茅台三季度营收创历史新高，净利润同比增长 15% | ✓ `{"sentiment": "positive", "confidence": 1.0, "keywords": ["茅台", "营收", "净利润", "增长"]}` | ✓ 同上 |
| 比亚迪召回 10 万辆电动车，涉及电池安全问题 | ✓ `{"sentiment": "positive", "confidence": 1.0, "keywords": ["比亚迪", "电池安全问题"]}` | ✓ 同上 |
| 央行维持 LPR 利率不变 | ✓ `{"sentiment": "positive", "confidence": 1.0, "keywords": ["央行", "LPR", "利率"]}` | ✓ 同上 |
| 宁德时代与宝马签订长期供货协议 | ✓ `{"sentiment": "positive", "confidence": 1.0, "keywords": ["宁德时代", "宝马"]}` | ✓ 同上 |
| 平安保险高管被调查，股价下跌 8% | ✓ `{"sentiment": "negative", "confidence": 0.75, "keywords": ["平安保险", "高管", "股价", "下跌"]}` | ✓ 同上 |

**5 条测试结果**

| 指标 | 裸 prompt | response_format |
|---|---|---|
| 合法 JSON | 5/5 (100%) | 5/5 (100%) |
| 有 sentiment 字段 | 5/5 (100%) | 5/5 (100%) |
| sentiment 值合法 | 5/5 (100%) | 5/5 (100%) |
| 有 confidence 字段 | 5/5 (100%) | 5/5 (100%) |
| 有 keywords 字段 | 5/5 (100%) | 5/5 (100%) |

**观察**：`response_format` 显著提升 JSON 合法率，但字段语义仍靠模型自觉；若需严格字段 schema，请用 `guided_json`（见第 5 节）。

## 核心结论

| 方式 | 特点 |
|---|---|
| 裸 prompt | JSON 语法偶尔错 / 字段拼错 / 正则枚举不符 |
| response_format | JSON 合法率接近满分，但字段语义仍会出错 |
| guided_json | 100% 满足完整 schema 结构（小模型从"不可用"变"可靠"） |

**关键 takeaway**：`guided_choice` / `guided_json` / `guided_regex` 能保证**格式/结构**合法，但不能保证**语义/内容**正确——语义正确率仍取决于模型本身的理解能力，约束解码解决的是"下游解析器崩溃"问题，不是"模型答错"问题。
