# 箱子管理

## 业务定位

箱子是库存管理中的物理包装单元，用于将多个散件商品组合为一个整体进行存储和流转。箱子通过唯一编码标识，支持入库、出库、拆箱等全生命周期管理。

## 核心概念

| 概念 | 说明 |
|------|------|
| 箱子 (StoreBox) | 物理包装单元，包含多个散件商品 |
| 箱子编码 (BoxCode) | 系统自动生成的唯一编码，格式为 `BOX` + 公司编码 + 仓库编码 |
| 箱子状态 | 未使用 → 已使用 → 已出库 / 已拆箱 |

## 业务规则

### 新增箱子

- 箱子编码由系统自动生成
- 仓库编码不能为空，否则报错
- 支持一次批量创建多个箱子（指定数量）
- 初始状态为"未使用"

### 删除箱子

- **只能删除未使用的箱子**
- 已使用、已出库的箱子均不允许删除

### 箱子出库

- 设置出库单号，状态变为"已出库"
- 箱子编码必须存在，否则报错

### 箱子入库

- 设置入库单号，状态变为"已使用"
- 箱子编码必须存在，否则报错

### 出库撤销（回退）

- 出库单号必须与箱子当前出库单号一致，否则报错"箱子与出库单不匹配"
- 撤销后清除出库单号，状态恢复为"已使用"

### 拆箱

- **前提条件**：箱子必须处于"已使用"状态
- 拆箱后状态变为"已拆箱"
- 拆箱后箱子内商品散放到指定库位
- 拆箱完成后通知下游系统

### 强制拆箱

- 与正常拆箱不同，强制拆箱直接指定出库单号
- 强制散放箱子内商品

## 业务流程

### 箱子状态机

```
              ┌──────────┐
              │  未使用   │ (初始状态)
              └────┬─────┘
                   │ [入库]
                   ▼
              ┌──────────┐
        ┌─────│  已使用   │─────┐
        │     └──────────┘     │
        │                      │
     [出库]                 [拆箱]
        │                      │
        ▼                      ▼
  ┌──────────┐          ┌──────────┐
  │  已出库   │          │  已拆箱   │
  └────┬─────┘          └──────────┘
       │
  [撤销出库]
       │
       ▼
  ┌──────────┐
  │  已使用   │
  └──────────┘
```

## 特殊场景处理

1. **拆箱事件发布**：拆箱操作完成后通知下游系统，确保事件发布失败不影响主业务
2. **箱子与套包对比**：箱子是物理包装，套包是逻辑组合；两者都有拆箱/拆包操作，但箱子拆箱会发布事件通知

## 操作指引

### 箱子列表

**查询条件**：公司、所属仓库（级联公司）、入库单号、出库单号、箱子编码（快捷搜索）、状态、备注。

**操作按钮**：
- **新增**：选择公司和仓库，指定创建数量
- **编辑**：需选中一条记录，仅可修改备注
- **拆箱**：需选中一条"已使用"状态的箱子，确认后拆箱
- **查看货品**：需选中一条"已使用"状态的箱子，弹窗展示箱内商品明细
- **删除**：需选中记录，只能删除未使用的箱子
- **打印条码**：需选中有箱子编码的记录

### 查看货品

弹窗展示箱子内的商品明细，包括：库位、供应商、业务单元、项目、厂家、品名、规格、货号、数量、生产日期、使用期限、批号、序列号、条码、货品类型。

## 数据模型

- 箱子 ID
- 箱子编码（系统生成）
- 所属公司
- 所属仓库
- 状态（未使用 / 已使用 / 已出库 / 已拆箱）
- 入库单号
- 出库单号

## 外部系统依赖

| 模块 | 用途 |
|------|------|
| 仓库管理 | 获取仓库编码用于生成箱码 |
| 库存服务 | 拆箱时散放商品到库位 |
| 库存查询 | 查询箱子内商品明细 |
