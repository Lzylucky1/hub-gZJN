# 套包管理

## 业务定位

套包是库存管理中的逻辑包装单元，将多个 SKU 按固定组合打包为一个销售/存储单元。与箱子不同，套包需要经历"赋码 → 扫码校验 → 提交"的完整流程，确保包内商品种类和数量与预设完全一致。

## 核心概念

| 概念 | 说明 |
|------|------|
| 套包 (StorePackage) | 逻辑包装单元，由多个商品按固定配比组成 |
| 套包编码 (PackageCode) | 系统生成，格式为 `TB` + 公司编码 + 仓库编码 |
| 套包货号 (ProductPackNo) | 套包对应的商品货号标识 |
| 套包状态 | 未使用 → 已使用 → 已出库 / 已拆包 |

## 业务规则

### 创建套包（赋码）

- 套包编码自动生成
- 仓库编码不能为空
- 支持一次批量创建多个套包
- 初始状态为"未使用"
- 创建时需关联：公司、仓库、厂家、品名

### 开始扫码校验

在正式扫码前进行前置校验：
1. 套包码必须存在且未使用
2. 目标库位必须存在
3. 套包与目标库位必须属于同一仓库
4. 套包与目标库位必须属于同一公司
5. 库位必须处于启用状态
6. 不能使用不合格库位
7. 套包只能使用"不校验"类型的库位
8. **医疗器械/非医疗器械隔离**：医疗器械库位只能入医疗器械商品，非医疗器械库位只能入非医疗器械商品

### 扫码结束校验

1. 验证源库位（扫码库位）存在且属于套包所在仓库
2. 验证目标库位（套包库位）存在且属于套包所在仓库
3. **跟台库位限制**：目标库位为跟台库位时报错"跟台库位只能用于跟台入库"
4. 校验条码对应库位的库存是否充足
5. 校验商品属性与目标库位是否匹配

### 提交套包

1. **状态校验**：只能提交未使用状态的套包
2. **库位验证**：所有涉及的库位存在并启用，目标库位不能是不合格品库位或停用状态
3. **数量完整性校验**：
   - 逐个核对每种商品的已扫数量与预设数量
   - 若套包中不存在某货号的商品，报错
   - 若数量不一致，报错显示应扫数量和实扫数量
4. **状态更新**：套包状态变为"已使用"，绑定库位
5. **库存操作**：将套包内商品组装入库存（散件减少、套包增加）

### 套包出库

- 套包必须处于"已使用"状态才能出库
- 设置出库单号

### 套包入库

- 套包必须处于"未使用"状态才能入库
- 设置入库单号

### 拆包

- 不能拆已拆的套包
- 只能拆"已使用"状态的套包
- 拆包后散放商品到指定库位

### 强制拆包

- 直接指定出库单号进行拆包
- 强制散放商品

### 出库撤销

- 出库单号必须匹配当前套包的出库单号
- 撤销后清除出库单号，状态恢复为"已使用"

### 删除套包

- 只能删除未使用的套包

## 业务流程

### 套包赋码完整流程

```
1. 创建套包（赋码）
   │  生成套包编码，状态=未使用
   │
2. 开始扫码校验
   │  校验套包、库位、公司、仓库一致性
   │  校验医疗器械/非医疗器械隔离
   │
3. 逐件扫码（每次扫一个商品条码）
   │  每次扫码后触发结束校验
   │  校验源库位库存、目标库位属性
   │
4. 提交套包
   │  校验每种商品已扫数量 = 预设数量
   │  状态 → 已使用
   │  库存组装：散件库存减少 → 套包库存增加
```

### 套包状态机

```
未使用 ──[提交]──→ 已使用 ──[出库]──→ 已出库
                      │                   │
                   [拆包]             [撤销出库]
                      │                   │
                      ▼                   ▼
                  已拆包               已使用
```

## 特殊场景处理

1. **箱码 vs 套包码**：箱码前缀"BOX"，套包码前缀"TB"，两者编码体系独立
2. **套包与箱子的出入库状态差异**：套包入库要求"未使用"状态，箱子入库直接设为"已使用"
3. **库位属性传递**：提交套包时，将目标库位的供应商和业务单元传递给套包明细

## 操作指引

### 套包列表

**查询条件**：套包编码（快捷搜索）、套包货号、入库单号、出库单号、公司、所属仓库（级联公司）、状态。

**操作按钮**：
- **套包赋码**：选择公司、仓库、套包货号，指定数量
- **拆包**：需选中一条"已使用"状态的套包
- **查看货品**：需选中一条"已使用"状态的套包
- **删除**：需选中记录，只能删除未使用的套包
- **打印条码**：需选中有套包编码的记录

### 套包赋码表单

**必填字段**：公司、所属仓库、套包货号、数量。

**套包货号选择联动**：选中货号后自动填充品名、厂家等信息。

### 查看货品

弹窗展示套包内的商品明细，与箱子货品查看一致。

## 数据模型

- 套包 ID
- 套包编码（系统生成）
- 套包货号
- 套包货号 ID
- 状态（未使用 / 已使用 / 已出库 / 已拆包）
- 所属公司
- 所属仓库
- 厂家
- 品名
- 套包库位
- 入库单号
- 出库单号

|------|------|
