# 库存基数管理

## 业务定位

库存基数是商品在特定维度下的理论库存基准值，通过与实际库存对比发现差异，实现库存预警功能。适用于药品、耗材等需要定期核对库存准确性的场景。

## 核心概念

| 概念 | 说明 |
|------|------|
| 库存基数 (StoreBasic) | 某商品在特定公司/仓库/项目/供应商/业务单元维度下的标准库存数量 |
| 库存预警 | 实际库存与基数偏差超过阈值时触发的告警 |
| 贸易类型 | 商品贸易分类 |
| 标记 | 基数数据的业务标记 |

## 业务规则

### 新增基数

- **唯一性校验**：以下 8 个字段组合必须唯一，不允许重复：
  - 公司、商品、贸易类型、标记、项目、仓库、业务单元、供应商
- 若已存在相同组合的记录，报错"已存在基本信息相同的数据"

### 批量更新基数

- 支持批量更新多条基数记录

### 批量删除基数

- 按选中记录批量删除

### Excel 导入基数

导入流程：
1. 上传 Excel 文件
2. 逐行解析并检查是否已有相同组合的记录
3. 若已存在，先删除旧记录再以新导入为准
4. 若不存在，创建新记录
5. 逐条进行唯一性校验
6. 批量保存

**错误处理**：导入失败时生成错误报告，可下载查看。

**导入提示**：如果存在同公司同项目同仓库同货号配置，导入会覆盖之前的配置。

### 基数预警查询

- 支持按公司、仓库、项目多维度查询
- 预警结果包含：商品名称、型号/剂型、基数、实际库存、偏差量
- 剂型描述优先展示，无剂型时展示型号
- 查询结果自动填充关联名称

### 基数数据导出

- 导出全量基数数据

### 预警数据导出

- 导出预警数据

## 业务流程

### 基数导入流程

```
1. 下载导入模板
   │
2. 填写基数数据（公司+商品+仓库+项目+供应商+业务单元+基数数量）
   │
3. 上传 Excel
   │
4. 系统逐行校验
   ├── 数据格式校验
   └── 重复性校验（8 字段组合唯一）
   │
5. 已存在记录 → 先删除旧记录
   不存在记录 → 直接创建
   │
6. 批量保存
   │
7a. 全部通过 → 成功
7b. 部分失败 → 生成错误报告 → 下载修正
```

### 预警查询流程

```
1. 设置查询条件（公司/仓库/项目）
   │
2. 系统查询基数配置
   │
3. 对比实际库存
   │
4. 计算偏差
   │
5. 返回预警列表
   │
6. 支持导出预警数据
```

## 特殊场景处理

1. **导入覆盖策略**：重复数据以新导入为准，先删除旧记录再创建新记录
2. **名称自动填充**：查询时自动填充关联名称（公司、仓库、项目、商品、厂家、业务单元、供应商）

## 操作指引

### 基数列表

**查询条件**：公司、仓库、业务单元、项目、品名（显示品名+厂家）、货号、货品类型、贸易类型、注册证、创建人、创建时间、编辑人、编辑时间、厂商。

**操作按钮**：
- **新增**：跳转到独立新增页面
- **编辑**：需选中记录，可修改最大基数和最小基数
- **删除**：需选中记录
- **导出数据**：按查询条件直接下载
- **导入清单**：下载模板、上传 Excel、下载错误报告

**批量编辑校验**：
- 选中多条记录时，需满足相同公司 + 业务单元 + 厂家 + 品名 + 货品类型
- 不满足时提示"批量编辑失败，相同公司+业务单元+厂家+品名+货品类型才支持批量编辑"
- 仅选中 1 条时直接允许编辑

**编辑弹窗**：仅修改最大基数和最小基数两个字段，提交前校验"最小基数不能大于最大基数"。

## 数据模型

**基数核心字段**：

- 公司
- 商品
- 贸易类型
- 标记
- 项目
- 仓库
- 业务单元
- 供应商
- 最大基数（库存上限）
- 最小基数（库存下限）

**预警输出字段**：

- 商品名称
- 型号/剂型
- 基数
- 实际库存
- 偏差量
