# vLLM vs Transformers：吞吐性能基准测试报告

## 测试环境

| 项目 | 配置 |
|------|------|
| GPU | NVIDIA L40S × 1（46GB 显存） |
| CUDA 驱动 | 535.161.08 / CUDA 12.6 |
| 模型 | Qwen2-0.5B-Instruct |
| 推理框架 | PyTorch 2.5.1、transformers 4.47.1、vLLM 0.7.3 |
| 测试请求数 | 50 条 prompt（短/中/长混合） |
| 最大生成长度 | 100 token |
| batch_size | 8（transformers 手动 batch） |

## 测试结果

| 模式 | 总耗时 (s) | QPS (req/s) | token/s | 相对 vLLM |
|------|-----------|-------------|---------|-----------|
| [A] transformers 串行 | 54.46 | 0.92 | 67 | 0.02× |
| [B] transformers batch=8 | 11.04 | 4.53 | 334 | 0.12× |
| [C] vLLM continuous batching | **1.28** | **39.19** | **2730** | 1.00× |

## 加速比

- vLLM 相对 **transformers 串行**：**42.7×** 加速
- vLLM 相对 **transformers batch=8**：**8.6×** 加速
- vLLM 生成吞吐是 transformers batch 的 **8.2×**

## 结论

vLLM 凭借 **PagedAttention**（KV cache 按 block 管理，消除内存碎片）和 **continuous batching**（不同长度请求动态组 batch，不等最长的），在 50 条混合长度 prompt 场景下，比 transformers 串行快 42.7 倍，比手动 batch=8 快 8.6 倍。
