# 文本分类方法对比报告

## 1. 核心指标对比

| 方法 | 准确率 | 训练成本 | 推理速度 | 数据需求 | 输出可控性 |
|------|--------|----------|----------|----------|------------|
| BERT Fine-tune | 0.4545 | 中等（CPU/GPU 均可） | ~5ms/条 | 需标注（53K） | 高（固定类别） |
| LLM Zero-Shot | 0.3600 | 无（零训练） | ~2.00s/条 | 零标注 | 低（需解析兜底） |
| LLM SFT (LoRA) | 0.5800 | 低（0.22% 参数） | ~0.06s/条 | 需标注（5K 即可） | 中（需解析） |

## 2. 方法选型建议

- **延迟敏感 + 有标注** → BERT Fine-tune（<20ms，100% 可控）
- **零标注 + 快速验证** → LLM Zero-Shot（分钟级上线）
- **大模型能力 + 少量标注** → LLM SFT + LoRA（单卡可训）
- **类别频繁变动** → LLM Zero-Shot / Few-Shot（改 prompt 即可）

## 3. 关键洞察

1. BERT 用 53K 数据达到 ~57-62% 准确率，是工程落地的首选（延迟低、成本低、输出稳）。
2. LLM Zero-Shot 无需训练即可达到可用水平，适合快速原型和零标注启动。
3. LLM SFT (LoRA) 仅用 5K 数据（BERT 的 9.4%）即可逼近 BERT 效果，体现大模型预训练知识的迁移效率。
4. 生成式分类（LLM）存在'无法解析'风险，生产系统需加兜底逻辑；判别式分类（BERT）无此问题。
